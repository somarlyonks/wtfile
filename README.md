# WTFile

An insane alternative to `pathlib.Path` and `path.py`.

## requirements

It's just an aggressive and insane option, it deponds on nothing but Python3.6.0+ that supports the powerful "path-like".

## usage

### callable string

```python
>>> from wtfile import F
>>> f = F('/home/sy', 'test.cc')
>>> f.ext
.cc
>>> f.ext('h')
/home/sy/test.h
>>> f.stem('name')
/home/sy/name.h
>>> f.name
name.h
>>> filepath = os.path.join('/home/sy', f.name)
/home/sy/name.h
```

### operations

```python
>>> f = F('/home/sy') / 'test.cc'
>>> f
/home/sy/test.cc
>>> f.name
test.cc
```

### oo && linkage

```python
>>> f = F('/home/sy', 'test.c')
>>> f.parent
/home/sy
>>> f.parent.children
[]
>>> f.touch()
>>> f.parent.children
['test.c']
>>> f.write('int main(void);').ext('h')
>>> f
/home/sy/test.h
>>> f.read()
'int main(void);'
```
